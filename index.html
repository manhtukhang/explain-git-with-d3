Trực quan hóa thao tác trên Git với D3

Trang web này được thiết kế để giúp bạn hiểu một số khái niệm cơ bản git bằng phương pháp trực quan. Đây là lần đầu tiên tôi sử dụng SVG và D3. Hi vọng bạn cảm thấy nó có ích!

Các tệp mà bạn commit sẽ không được ghi lại bởi trang này. Bạn có thể coi như đang chạy trên máy ảo. Nếu cần thêm thông tin về cách add/stage, vui lòng đọc Git Basics.

Mỗi lệnh sẽ được chia nhỏ vào các "hộp cát" sau:

Các lệnh cơ bản

git commit git branch

 

git checkout git checkout -b

Hủy Commit

git reset git revert

Kết hợp nhánh

git merge git rebase

Máy chủ từ xa

git fetch git pull

 

git push

Tôi sẽ bỏ qua việc hướng dẫn bạn cách thêm tệp vào trước khi commit. Chúng ta giả sử bạn đã biết cách làm điều đó như thế nào. Nếu không bạn hãy đọc các hướng dẫn cơ bản về Git trước đã.

Giả sử bạn đã có các tệp được `staged' cho `commit', hãy gõ git commit vài lần để xem nó hoạt động ra sao.

git branch tênnhánh sẽ tạo ra 1 nhánh có tên "tênnhánh". Tạo nhánh chỉ tạo ra 1 thẻ trỏ vào `commit' hiện tại đang được `checkout'.

Nhánh có thể xóa bằng lệnh git branch -d tênnhánh.

Gõ git commit và git branch cho đến khi bạn hiểu về khái niệm này.

git checkout có nhiều công dụng, nhưng chính vẫn là để chuyển giữa các nhánh.
Ví dụ, để chuyển từ nhánh `master' đến nhánh `dev', Gõ git checkout dev. Sau đó, nếu gõ `git commit', để ý xem nó sẽ đi đâu. Thử đi.

Ngoài việc chuyển giữa các nhánh, bạn cũng có thể chuyển giữa các `commit' riêng rẽ. Thử đi.
Tạo 1 commit rồi gõ git checkout bb92e0e và xem có gì thay đổi.

Gõ git commit, git branch, và git checkout vài lần để hiểu rõ hơn.

Bạn có thể gộp git branch và git checkout vào chung 1 lệnh bằng cách gõ git checkout -b tênnhánh. Nó sẽ tạo ra 1 nhánh nếu chưa có và sẽ chuyển qua nhánh đó ngay lập tức.

git reset sẽ chuyển con trỏ HEAD và nhánh hiện tại lại bất cứ chỗ nào bạn chỉ định, đồng thời bỏ qua bất kì commit nào còn lại ở phía sau. Sẽ hữu ích nếu bạn muốn hủy bỏ 1 commit mà bạn không cần đến nữa.

Lệnh này thông thường được sử dụng với 3 tùy chọn sau đây: "--soft", "--mixed", và "--hard". Tùy chọn `soft' và `mixed' bạn đọc thêm ở đây. Trang này không thể hỉên thị 2 tùy chọn trên thực hiện ra sao, chỉ có "--hard" là được sử dụng ở đây.

Tham chiếu "HEAD^" thường được sử dụng kèm với lệnh này. "HEAD^" có nghĩa "commit trước HEAD". "HEAD^^" có nghĩa "2 commit trước HEAD", và tương tự...

Chú ý: bạn không được sử dụng git reset để hủy bỏ các commit đã được đẩy và gộp lên kho chứa chung. Điều này có thể khiến kho chứa cục bộ của bạn vượt ra khỏi việc đồng bộ hóa với kho chứa chung. Chỉ làm khi bạn chắc chắn về hậu quả của việc đó.

Để hủy bỏ những `commit' đã đẩy lên kho chứa chung và chia sẻ với mọi người, chúng ta không sử dụng git reset. Thay vào đó, ta phải sử dụng lệnh git revert.

git revert tạo ra 1 `commit' mà hủy tất cả công việc đã hoàn thành trong `commit' bạn muốn `revert'.

git merge sẽ tạo ra 1 `commit' với 2 cha. Kết quả là `commit' đó sẽ chứa cả kết quả của 2 nhánh cha.

Nếu không có sự khác biệt giữa 2 `commit', git sẽ thực hiện phương pháp gộp "chuyển tiếp nhanh".
Để xem điều gì xảy ra, hãy `checkout' nhánh 'ff' rồi gõ git merge dev.

git rebase sẽ đưa các commit trên nhánh này và "chuyển" chúng nên "base" mới của chúng sẽ là điểm mà bạn chỉ định.

Bạn nên chú ý đến các `commit ID' trong vòng tròn khi chúng di chuyển lúc bạn làm bài tập này.

Lý do mà tôi để từ "chuyển" trong dấu "" là bởi vì vịêc này thực chất sẽ tạo ra các commit mới với ID hoàn toàn khác các commit cũ, đồng thời bỏ lại các commit cũ. Chính vì lý do này, bạn sẽ không muốn (chính xác hơn là không được) `rebase' các commit mà đã đẩy lên kho chung và chia sẻ với mọi người.

git fetch sẽ cập nhật tất cả "các nhánh đang theo dõi từ xa" trong kho chứa cục bộ. Các nhánh đang theo dõi từ xa được đánh dấu trong thẻ màu xám.

A git pull là 2 bước trong đó bước thứ nhất là git fetch, và rồi sau đó là git merge của nhánh đang theo dõi từ xa liên quan đến nhánh hiện tại của bạn. Nếu bạn không có co nhánh nào hiện tại, tiến trình sẽ dừng sau khi "fetch" (lấy thông tin).

Nếu đối số "--rebase" được đưa vào bằng cách gõ git pull --rebase, thì bước thứ 2 sẽ `rebase' thay vì `merge'. ều này có thể đặt mặc địinh trong cấu hình bằng cách gõ: git config branch.BRANCHNAME.rebase true.

A git push sẽ tìm các commit trong kho cục bộ của bạn có nhưng trên kho chứa chung chưa có, và sẽ đẩy chúng lên kho chứa chung đó.

Mặc định, `push' sẽ tạo ra gộp chuyển tiếp nhanh (fast-forward merge) trên kho chung. Nếu có bất kì khác biệt nào giữa kho bạn và kho chung thì sẽ bị từ chối. Trong trường hợp này, bạn phải `pull' trước rồi sau đó mới `push' lại lần nữa.

Một ví dụ về việc sư dụng git reset là hoàn toàn khôi phục kho chứa của bạn về trạng thái gốc.
Bạn có thể gõ git reset origin/master.

Chú ý: điều này sẽ không xóa các tệp chưa đánh dấu, bạn sẽ phải xóa chúng riêng với lệnh git clean -df.

Dưới đây là một tình huống mà bạn đang làm việc trong 1 nhánh cục bộ của mình. Bạn muốn nhận được code mới nhất từ nhánh `master' trên máy chủ gốc. Để cập nhật nhánh cục bộ, bạn có thể thực hiện mà không phải chuyển nhánh!

Đầu tiên hãy git fetch, sau đó git rebase origin/master!

git branch -d được sử dụng để xóa 1 nhánh. Tôi đã tạo ra các nhánh có sẵn dưới đây cho bạn.

Làm bất cứ điều gì trong "sân chơi này" đi!
